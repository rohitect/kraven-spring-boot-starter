Plugin System Overview

Kraven UI features a powerful plugin system that allows you to extend its functionality with custom plugins. The plugin architecture is designed to be modular, allowing features to be added or removed as needed.

Key aspects of the plugin system include:

  • Dynamic Registration - Plugins register with the core at runtime
  • UI Integration - Plugins automatically appear in the UI when registered and running
  • Independent Lifecycle - Start, stop, and manage plugins independently
  • Configurable - Each plugin has its own configuration options

Kafka Plugin

The Kafka Plugin provides comprehensive Kafka management capabilities directly within your application. No more switching between tools or using external Kafka management interfaces - everything you need is right here in your application.

Key Features:

  • Topic Browser - View all topics and their configurations
  • Message Producer - Send messages to topics with a simple UI
  • Message Consumer - View messages in topics with filtering and pagination
  • Live Streaming - Watch messages arrive in real-time
  • Consumer Group Monitoring - Track consumer groups and their lag
  • Partition Details - View detailed partition information
  • Smart Loading - Features a sarcastic, funny loading animation when refreshing the page

Installation:

<dependency>
    <groupId>io.github.rohitect</groupId>
    <artifactId>kraven-ui-kafka-plugin</artifactId>
    <version>1.0.7</version>
</dependency>

Configuration:

# Enable the plugin system
kraven.ui.plugins.enabled=true

# Enable the Kafka plugin
kraven.ui.kafka.enabled=true

# Configure message browsing
kraven.ui.kafka.message-limit=100

# Enable/disable streaming
kraven.ui.kafka.streaming-enabled=true

# Enable/disable message production
kraven.ui.kafka.message-production-enabled=true

# Configure Kafka clusters (optional - will use Spring Kafka's configuration by default)
kraven.ui.kafka.clusters[0].id=local
kraven.ui.kafka.clusters[0].name=Local Cluster
kraven.ui.kafka.clusters[0].bootstrapServers=localhost:9092

Mock Server Plugin

The Mock Server Plugin provides a configurable mock server for integration testing and development. It's perfect for testing your application without depending on external services or for developing frontend applications while backend services are still in development.

Key Features:

  • Configurable Mock Server - Starts on port 11000 (or custom port)
  • JSON Configuration - Define mock endpoints and responses via JSON
  • Multiple Response Variations - Support for multiple response variations per endpoint
  • Runtime Switching - Switch between different mock responses via the UI
  • Request History - Track and visualize request history
  • Response Delay Simulation - Simulate network latency for realistic testing
  • Dynamic Response Generation - Generate responses based on request data

Installation:

<dependency>
    <groupId>io.github.rohitect</groupId>
    <artifactId>kraven-ui-mock-server-plugin</artifactId>
    <version>1.0.8</version>
</dependency>

Configuration:

# Enable the plugin system
kraven.ui.plugins.enabled=true

# Mock Server plugin configuration
kraven.ui.plugin.mock-server.enabled=true
kraven.ui.plugin.mock-server.auto-start=false  # Set to true to start the server automatically
kraven.ui.plugin.mock-server.port=11000
kraven.ui.plugin.mock-server.host=localhost
kraven.ui.plugin.mock-server.config-path=classpath:mock-server/config.json
kraven.ui.plugin.mock-server.auto-reload=true
kraven.ui.plugin.mock-server.reload-interval-ms=5000
kraven.ui.plugin.mock-server.default-delay-ms=0
kraven.ui.plugin.mock-server.default-template-engine=simple  # Options: simple, handlebars
kraven.ui.plugin.mock-server.enable-advanced-matching=true
kraven.ui.plugin.mock-server.enable-dynamic-responses=true

Actuator Insights Plugin

The Actuator Insights Plugin provides a powerful visualization and monitoring layer for Spring Boot Actuator endpoints, making it easy to monitor and manage your application's health, metrics, configuration, and thread activity.

Key Features:

  • Real-time Visualization - Interactive dashboards for application health, metrics, and system resources
  • Smart Auto-detection - Automatically detects Spring Boot Actuator in your application
  • Comprehensive Thread Analysis - Thread state distribution, deadlock detection, and thread pool insights
  • Advanced Environment Visualization - Property source exploration with search highlighting
  • Historical Data Analysis - Track metrics over time with trend visualization
  • Auto-Configuration Analysis - Understand why Spring Boot auto-configurations were applied or skipped
  • Application Log Viewer - Stream and analyze application logs with advanced filtering and search
  • Tab-based Interface - Health & Status, Metrics & Performance, Environment & Config, Beans & Dependencies, Thread Analysis, Conditions Report, Application Logs

Installation:

<dependency>
    <groupId>io.github.rohitect</groupId>
    <artifactId>kraven-ui-actuator-insights-plugin</artifactId>
    <version>1.0.8</version>
</dependency>

Configuration:

# Enable the plugin system
kraven.ui.plugins.enabled=true

# Actuator Insights plugin configuration
kraven.plugins.actuator-insights.enabled=true
kraven.plugins.actuator-insights.auto-detect=true
kraven.plugins.actuator-insights.base-url=http://localhost:8080
kraven.plugins.actuator-insights.context-path=
kraven.plugins.actuator-insights.data-collection.interval=15s
kraven.plugins.actuator-insights.data-collection.retention-period=1h
kraven.plugins.actuator-insights.endpoints.include=*
kraven.plugins.actuator-insights.endpoints.exclude=heapdump,shutdown
kraven.plugins.actuator-insights.sensitive-data.mask-sensitive-values=true
kraven.plugins.actuator-insights.sensitive-data.sensitive-patterns=password,passwd,secret,credential,token,key,auth,private,access

# Spring Boot Actuator configuration (recommended)
management.endpoints.web.exposure.include=*
management.endpoint.health.show-details=always
management.endpoint.env.show-values=always  # For Spring Boot 3 to show environment values

Plugin Development

Kraven UI provides a Plugin SDK that allows you to develop your own plugins. The SDK provides a clean API for registering new features and UI components.

Getting Started with Plugin Development:

  1. Add the Plugin SDK dependency: 
    <dependency>
    <groupId>io.github.rohitect</groupId>
    <artifactId>kraven-ui-plugin-sdk</artifactId>
    <version>1.0.6</version>
</dependency>
  2. Create a plugin class: 
    // Plugin class example
class MyPlugin {
    // Initialize method
    initialize(context) {
        // Register navigation items, controllers, services, etc.
    }

    // Start method
    start() {
        // Start your plugin
    }

    // Stop method
    stop() {
        // Stop your plugin
    }
}
  3. Create a plugin.properties file:

    Create a file at src/main/resources/META-INF/kraven/plugin.properties with the following content:

    mainClass=com.example.MyPlugin
id=my-plugin
name=My Plugin
version=1.0.0
description=My custom plugin for Kraven UI
provider=Your Name
  4. Build and package your plugin: 
    mvn clean package

Plugin Configuration

The plugin system can be configured using the following properties: 

# Enable or disable the plugin system
kraven.ui.plugins.enabled=true

# Enable or disable auto-discovery of plugins
kraven.ui.plugins.auto-discovery=true

# The base packages to scan for plugins
kraven.ui.plugins.base-packages=io.github,com,org,net
